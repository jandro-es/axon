# Proactive layer — daily briefing + resurfacer — design

**Date:** 2026-07-04
**Status:** approved (brainstormed + user-approved)
**Traces to:** new FR-88…FR-90 (docs/03), ADR-007 (chokepoint), ADR-011 (SessionStart injection), ADR-015 (local routing — the narrative rides it), ADR-017 (budget_tokens enforcement), new ADR-018

## Goal

Make AXON come to the user instead of only maintaining the vault: a morning
**briefing** written into the daily note (what changed yesterday, what's
pending, budget state, plus a short narrative), pointed to by every Claude
session; and a **resurfacer** that proposes connections between what you're
working on now and dormant notes you wrote months ago — the
spaced-serendipity function of a real second brain.

## Background / constraints

- Heartbeat already demonstrates the write pattern: ensure `Daily/<date>.md`
  (dailyStub) → `vault.Patch(ctx, path, "<block>", content)`; distinct
  `axon:*` blocks coexist in one note, and Patch creates the block if absent.
- SessionStart (`internal/hooks`) injects budget/inbox/review-queue lines +
  the token-bounded identity block; every added line must be deterministic,
  no-model, and degrade to nothing on error.
- The `notes` table has day-granular `updated` (frontmatter or file mtime at
  last reindex); there is **no recency query yet**.
- Note-to-note similarity exists only as the dashboard's **unexported**
  `similarityEdges` (mean of chunk vectors per note, brute-force cosine,
  threshold 0.75, top-3 per note, skip above 1500 notes).
- Link-suggester has **no persistent already-proposed memory** — only its
  vectors-count cursor prevents re-proposing. The resurfacer must do better.
- `automations.<name>.budget_tokens` is enforced at runtime since ADR-017;
  the routine tier is local-routable since ADR-015.

## Decisions (user-approved)

1. **Briefing = deterministic facts + one-shot routine-tier narrative.**
   Facts are assembled in Go (zero tokens) and always written; a short
   narrative paragraph is generated by a single `runModel` call on the
   routine tier — local-routable (ADR-015), budget-capped (ADR-017), and on
   defer/denial the briefing degrades to facts-only rather than failing.
   *Rejected:* facts-only (reads like a status table); agentic synthesis
   (daily frontier-tier spend for facts the queries already know — the
   agentic win is thin here, unlike digest).
2. **SessionStart injects a one-line pointer**, not the block: `- Briefing:
   Daily/<date>.md (axon:briefing)`, emitted only when today's block exists.
   *Rejected:* injecting the bounded block (hundreds of tokens every session
   for content one read away).
3. **Resurfacer is no-model**, weekly by default, with **persistent proposal
   memory** so a pair is never re-proposed.

## Design

### The `briefing` automation (`internal/automations/proactive.go`)

- Registry name `briefing`; `Essential() == false`.
- **DetectChange:** cursor `briefing:<YYYY-MM-DD>` (UTC, `today(rc)`) —
  changed iff the cursor differs, i.e. at most one briefing per day;
  `catch_up: run-once` writes this morning's briefing after downtime.
- **Run:**
  1. Assemble **facts** (all deterministic):
     - Notes changed yesterday/today: new `db.NotesUpdatedSince(ctx, q,
       sinceDate string) ([]NoteRow, error)` (`updated >= ?`, capped list,
       newest first) — rendered as `[[wikilinks]]` (up to ~10, then "+N more").
     - New sources since yesterday: `db.CountSourcesSince`.
     - Automation activity: `db.RecentRuns` filtered to the last day —
       counts by status, failures named.
     - Review queue: pending checkbox count (same parse as the hook).
     - Budget: `Manager.Status` (day/week %, guard state).
  2. **Narrative** (skippable): one `runModel` call, `ModelKey: "routine"`,
     operation `automation.briefing`, the facts passed as fenced DATA with
     the standard treat-as-data framing, asking for 2–4 sentences of
     orientation ("yesterday centered on…; watch out for…"). On
     `deferred` (budget) the narrative is omitted with a `(narrative skipped:
     budget)` marker line — the run still succeeds.
  3. Write `axon:briefing` in `Daily/<date>.md` (heartbeat's ensure→Patch
     pattern): narrative paragraph (if any) followed by the facts list and a
     generated-at stamp.
  - Dry-run: report the facts summary + narrative estimate, write nothing
    (`runModel` is already Authorize-only under dry-run).
- Starter config: `briefing: {enabled: true, schedule: "0 6 * * *", model:
  routine, budget_tokens: 40_000, catch_up: run-once}`.

### SessionStart pointer (`internal/hooks/hooks.go`)

After the review-queue line in `sessionStart`: if `Daily/<today>.md` exists
and contains an `axon:briefing` block (extractBlock-style check on the body),
append `- Briefing: Daily/<date>.md (axon:briefing)`. Deterministic, no
model, any error → no line. No new config (the line is ~10 tokens; the
existing `memory.inject` toggle is unrelated and untouched).

### The `resurfacer` automation (`internal/automations/proactive.go`)

- Registry name `resurfacer`; `Essential() == false`; **no model call**.
- **New db surface (`internal/db`):**
  - `NotesUpdatedSince(ctx, q, sinceDate string) ([]NoteRow, error)` —
    shared with briefing.
  - Extract the dashboard's mean-vector machinery into an exported
    `db.NoteMeanVectors(ctx, q) (map[int64][]float32, error)` +
    `db.NoteVectorSimilar(recent map[int64][]float32, all
    map[int64][]float32, threshold float64, perSource int)`-style helper —
    concrete shape decided in the plan, with `similarityEdges` refactored to
    use the same primitives so graph and resurfacer share one
    implementation (`cosine`, `DecodeVector`, `simThreshold = 0.75`).
- **Selection:** recent = notes `updated` within the last 7 days; dormant =
  notes `updated` at least 90 days ago (constants in code; both sets bounded
  — recent ≤ 50, dormant unbounded but the mean-vector map is skipped above
  the existing 1500-note ceiling, mirroring the graph guard).
- **DetectChange:** cursor `resurface:<vectors-count>:<ISO week>` — reruns
  when embeddings changed or a new week started; skips otherwise.
- **Run:** cosine each recent note's mean vector against dormant notes'
  mean vectors; keep pairs ≥ 0.75; exclude pairs already linked (reuse
  `linkTargets` on the recent note's body) and pairs in the **proposal
  memory** (`automation_state` row `resurfacer:proposed`, a JSON set of
  `"recentPath\x00dormantPath"` keys — unordered canonical form — capped at
  the newest 500 entries); take the top 5 by similarity; append to
  `.axon/review-queue.md`:
  `- [ ] resurface [[dormant]] — related to recent [[touched]] (sim 0.82, dormant since 2026-01-14)`;
  persist the updated proposal memory (non-dry runs only).
- Starter config: `resurfacer: {enabled: true, schedule: "0 7 * * 1", model:
  none, budget_tokens: 0}`.

### Testing

- db: `NotesUpdatedSince` table-driven (boundary dates, ordering, cap);
  mean-vector/similarity helpers against hand-computed vectors; graph
  `similarityEdges` regression (same edges before/after the refactor).
- briefing: facts assembly against a seeded temp vault + DB (runs, sources,
  review queue, budget via fake manager); narrative present when the fake
  responds / absent-with-marker when deferred; once-per-day gate; dry-run
  writes nothing; block lands beside `axon:heartbeat` without clobbering.
- hook: pointer line present only when today's block exists; absent on
  missing note/block; errors degrade silently.
- resurfacer: pair selection with seeded vectors (recent×dormant only —
  recent×recent pairs excluded), threshold, already-linked exclusion,
  proposal-memory exclusion across two runs, cap, review-queue format,
  dry-run, change gate.
- Registration tests mirror capture's (registry count 13, catalog purposes,
  schedulables).

## Trade-offs accepted

- The `updated` column is day-granular and only as fresh as the last
  reindex — "changed yesterday" means "reindexed as changed", which the
  nightly knowledge-reindex (and any capture/ingest activity) keeps honest
  enough for a briefing.
- Brute-force mean-vector cosine is O(recent × dormant); bounded by the
  recent cap (50), the 1500-note ceiling, and weekly scheduling.
- The briefing narrative spends routine-tier tokens daily unless routed
  locally; `budget_tokens` and the facts-only degradation bound the cost.

## Out of scope

- A weekly-review automation; spaced-repetition scheduling algorithms.
- Resurfacing UI beyond the review queue; auto-linking resurfaced pairs.
- Calendar or other external-source awareness in the briefing.
- Retrofitting proposal memory into link-suggester (noted in ADR-018 as a
  follow-up).
